# Consult Note Handoff - 2026-07-08

## Current Status

- WPF + SQLite 로컬 앱 MVP는 실제 사용 가능한 수준까지 정리되었습니다.
- 다른 PC에서 self-contained 배포본 실행 테스트가 확인되었습니다.
- 파일 사이드바는 기본 닫힘 상태이며, 필요할 때 `파일 >` 버튼으로 열 수 있습니다.
- 고객 파일은 DB에 메타데이터를 저장하고 실제 파일은 `storage/customers/{customerId}/files/`에 복사합니다.
- 새로 추가되는 고객 파일 저장명은 `고객명_파일유형_순번.확장자` 형식입니다.
- 견적 파일은 메모가 있으면 `고객명_견적_순번_메모.확장자` 형식으로 저장됩니다.
- 파일 순번은 사용자가 직접 입력하지 않고 DB의 같은 고객/같은 파일 유형 기준으로 자동 증가합니다.
- 상담 양식 복사는 클립보드 복사와 `logs/exports/` txt 보조 저장을 함께 수행합니다.

## Recent Decisions

- 상담 기록 export는 PDF보다 txt를 우선합니다.
- 이유: 회사 사이트에 붙여넣기 쉽고, 파일이 가볍고, 드래그/복사 흐름에 더 잘 맞습니다.
- 고객별 전체 상담 기록 txt 내보내기는 P3.5 후속 작업으로 남겨두었습니다.
- `scripts/publish.ps1`은 self-contained publish 후 `dist/`에 zip까지 생성하므로 P3.6 배포용 압축 자동화는 완료로 봅니다.
- 백업은 당장 자동화하지 않고, 우선 `consultnote.db`와 `storage/`를 함께 보관하는 수동 백업으로 충분합니다.
